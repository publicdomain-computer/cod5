General text formatting

Statements inside blocks shall be indented by a tab.
Tabs shall be 8 blank characters wide.
Text lines shall not have more than 80 characters.
Source files and directories names shall contain only ASCII characters.

Include files

The first line of an include file shall begin with a multi-lines comment. It shall contain the  CVS/SVN "$Id: $" string, the public domain notice and optionally a description of the file's content or short citation.

Then an #ifndef/#define sequence shall be inserted to prevent multiple inclusion of the file.
The defined macro name shall begin with the directory relative path and ending with the include file name. The slash or backslash characters are replaced by two underlines and the dot characters are replaced by one underline.

Flowing are the #include directives, the definition of macros, typedefs, structures declarations, exported functions and exported variables.

At the end of the file there shall be the #endif directive.

Include file example:

/* "$Id: $"
 *
 *                          10 September 2009 PUBLIC DOMAIN
 *             The author disclaims copyright to this source code.
 */

#ifndef LANG__STRING_H
#define LANG__STRING_H

#include <lang/Object.h>

#define STR(s, t) { \
    s.size = 0; \
    s.cstr = (utf8*)t; \
}

typedef struct String__class_struct {
    utf8 *__name__;
    uint32 __size__;
    struct Object__class_struct *super;
} String__class;

EXPORT void String__add(Ex **e__, String *self, String *in);

EXPORT String__class String__;

#endif // LANG__STRING_H

Source files

The header comment shall be the same as the corresponding include file.
